神圣经典翻译流水线，结合人工审核治理

项目概述

问题：目前27位译者的团队每年大约只能完成1本书的翻译。协调审核、编辑和校对的工作依赖电子邮件和共享文档，没有结构化的流程、没有质量核查清单、也没有统一的术语库。翻译质量很大程度上取决于个人判断，难以维持巴哈伊圣作所要求的三项标准：准确、文风优美、风格一致。

解决方案：巴哈伊华语翻译工作台是一个结合AI辅助与人工治理的翻译流水线。它实施5个阶段的工作流程 — 翻译、审核、编辑、排版、校对 — 由AI处理繁重工作，但人工审核员在第2阶段（审核）和第5阶段（校对）做出最终决定。

三项翻译标准

以学习材料《关于语言和翻译指引选编》为基础，该选编涵盖巴哈伊圣作中9个翻译指引章节，工作台在每个审核关卡执行三项质量标准：

  • 准确：忠实于圣典原文的含义
  • 文风优美：措辞典雅庄重，符合神圣经典的风格
  • 风格一致：整部作品的术语、语调和语体保持统一

应用功能

5阶段流水线

结构化工作流程：翻译 → 审核 → 编辑 → 排版 → 校对。每个阶段有指定的角色和明确的输入/输出要求。

姓名登录

简单的姓名登录，无需密码。每位用户被分配一个角色（协调员、审核员、译者、术语专员或最终审阅人），决定其权限和职责。

单人 / 双人审核员模式

短篇文档由单一审核员负责第2和第5阶段。长篇作品则采用双人审核员模式：审核员A负责第2阶段（审核），审核员B负责第5阶段（校对）。

三项标准核查清单

在第2阶段，审核员根据三项标准审查AI翻译：准确、文风优美、风格一致。每项必须明确通过后方可推进。

多模型选择器

协调员为流水线的每个阶段选择使用哪个 LLM 模型。支持 Claude Sonnet、GPT-4、Gemini 和 DeepSeek — 让团队能够比较并选择最适合的模型。

流水线仪表板

可视化展示所有进行中的翻译流水线，显示当前阶段、指派的审核员、已选模型以及每篇文档的进度状态。

应用内术语编辑器

术语专员在应用内维护一份已批准的巴哈伊术语翻译词汇表。该词汇表会注入到AI提示中，确保所有翻译的术语一致性。

团队管理

协调员管理团队成员、分配角色并将人员配置到特定的流水线阶段。支持27位译者在多个角色中的完整组织架构。

AI 整合

多模型适配器

统一的适配层支持 Claude Sonnet 4.6、GPT-4、Gemini 和 DeepSeek。协调员在流水线层级为每个阶段设定模型，实现翻译质量的并排比较。

提示中的三项标准框架

每个AI提示都包含明确指示，要求评估和优化准确性、文风优美和风格一致性 — 这三项源自巴哈伊圣作的标准。

术语词汇表注入

已批准的巴哈伊术语词汇表会自动注入到AI提示中，确保关键的宗教和哲学术语在所有流水线阶段中翻译一致。

第4阶段排版验证

在排版阶段，AI验证华语神圣经典特有的格式、标点和版面规范，在最终校对前标记问题。

治理模式

单人审核员模式

适用于较短的文档，审核员A同时负责第2阶段（审核）和第5阶段（校对）。由一人保持对整个翻译的完整把握，提供端到端的质量控制。

双人审核员模式

适用于较长的作品，审核员A负责第2阶段（审核），审核员B负责第5阶段（校对）。分散工作量的同时确保最终关卡有全新的视角。

团队架构

27人的翻译团队按专业角色组织：

  • 18位审核员：资深译者，在第2和第5阶段审核和批准翻译
  • 4位术语专员：维护词汇表，确保术语一致性
  • 3位协调员：设置流水线、指定模型、管理团队分配
  • 2位最终审阅人：在出版前提供最后一道质量关卡

预期成果

通过结构化的流水线和AI辅助，团队可以运行4至6条并行流水线，将产出从每年1本书提升到预计每年4至6本书。

Docker Google Cloud Run

系统设计

  • FastAPI 后端：处理所有 API 路由、流水线编排和 LLM 调用
  • SQLite 数据库：存储用户、流水线、词汇表术语和翻译历史
  • 多模型适配器：Claude、GPT-4、Gemini 和 DeepSeek API 的统一接口（公共演示中仅 Claude 启用；自行部署时可启用其他模型）
  • 静态前端：由 FastAPI 直接提供 Vanilla HTML/JS，无需构建步骤
  • Docker 容器：在 Google Cloud Run 上进行单容器部署。公共演示采用"自带密钥"模式——访客在浏览器中提供自己的 Anthropic API 密钥，服务器不保存任何密钥

安装指南

试用在线演示

  • 一把免费的 Anthropic API 密钥——请前往 console.anthropic.com 注册（约需 2 分钟；Anthropic 会为新账户提供少量免费额度）
  • 打开在线应用，将密钥粘贴到页面顶部的 API Key 栏，并以 Zhang Wei（协调员）身份登录，即可体验完整流程
  • 您的密钥仅保存在您的浏览器中（localStorage），并直接发送到 Anthropic——我们的服务器绝不会存储它

自行部署（可选）

如果您希望自行部署、启用全部四个 LLM 供应商、并关闭"自带密钥"提示，可克隆代码库并通过 .env 提供服务器端密钥：

# 克隆代码库 git clone https://github.com/lyven81/bahai-translation-workbench.git cd bahai-translation-workbench # 安装依赖 pip install -r requirements.txt # 设置环境变量 cp .env.example .env # 在 .env 文件中添加你的 API 密钥（ANTHROPIC_API_KEY、OPENAI_API_KEY、GOOGLE_API_KEY、DEEPSEEK_API_KEY） # 运行应用 python app.py
