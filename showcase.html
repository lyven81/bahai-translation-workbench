Bahai Chinese Translation Workbench

Sacred text translation pipeline with human governance

Project Overview

Problem: A team of 27 translators currently produces approximately 1 book per year using a manual translation process. Coordination across reviewers, editors, and proofreaders is done through email and shared documents, with no structured pipeline, no quality checklist, and no centralized glossary. Translation quality depends heavily on individual judgment, making it difficult to maintain the three standards required by the Baha'i Writings: accuracy, beauty of language, and stylistic consistency.

Solution: The Bahai Chinese Translation Workbench is an AI-assisted translation pipeline with human governance built into every critical stage. It implements a 5-stage workflow — Translation, Review, Editing, Typesetting, and Proofreading — where AI handles the heavy lifting, but human governors make the final call at Stage 2 (Review) and Stage 5 (Proofread).

Three Translation Standards

Grounded in the study material "关于语言和翻译指引选编" (Compilation on Language and Translation Guidelines), which covers 9 sections of translation guidance from the Baha'i Writings, the workbench enforces three quality standards at every review checkpoint:

  • Accuracy (准确): Faithful to the original meaning of the sacred text
  • Beauty of Language (文风优美): Elegant and dignified expression befitting sacred scripture
  • Stylistic Consistency (风格一致): Uniform terminology, tone, and register across the entire work

Application Features

5-Stage Pipeline

Structured workflow: Translation → Review → Editing → Typesetting → Proofread. Each stage has a designated role and clear input/output expectations.

Name-Based Login

Simple login by name — no passwords. Each user is assigned a role (coordinator, governor, translator, terminology specialist, or final reviewer) that determines their access and responsibilities.

Single / Dual Governor Model

For short documents, a Single Governor handles both Stage 2 and Stage 5. For longer works, the Dual Governor model assigns Governor A at Stage 2 (Review) and Governor B at Stage 5 (Proofread).

Three-Standard Checklist

At Stage 2, the governor reviews the AI translation against the three standards: accuracy, beauty of language, and stylistic consistency. Each must be explicitly approved before advancing.

Multi-LLM Selector

The coordinator selects which LLM model to use at each stage of the pipeline. Supports Claude Sonnet, GPT-4, Gemini, and DeepSeek — allowing the team to compare and choose the best model per task.

Pipeline Dashboard

Visual overview of all active translation pipelines showing current stage, assigned governor, selected model, and progress status for each document in the system.

In-App Glossary Editor

Terminology specialists maintain an in-app glossary of approved translations for key Baha'i terms. The glossary is injected into AI prompts to ensure consistent terminology across all translations.

Team Management

Coordinators manage team members, assign roles, and allocate people to specific pipeline stages. Supports the full organizational structure of 27 translators across multiple roles.

AI Integration

Multi-LLM Adapter

A unified adapter layer supports Claude Sonnet 4.6, GPT-4, Gemini, and DeepSeek. The coordinator sets the model for each stage at the pipeline level, enabling side-by-side comparison of translation quality.

Three-Standard Framework in Prompts

Every AI prompt includes explicit instructions to evaluate and optimize for accuracy, beauty of language, and stylistic consistency — the three standards derived from the Baha'i Writings.

Terminology Glossary Injection

The approved glossary of Baha'i terms is automatically injected into AI prompts, ensuring that key religious and philosophical terms are translated consistently across all pipeline stages.

Stage 4 Typesetting Validation

At the Typesetting stage, the AI validates formatting, punctuation, and layout conventions specific to Chinese-language sacred texts, flagging issues before the final proofread.

Governance Model

Single Governor

For shorter documents, Governor A owns both Stage 2 (Review) and Stage 5 (Proofread). This provides end-to-end quality control by a single person who maintains full context of the translation.

Dual Governor

For longer works, Governor A handles Stage 2 (Review) and Governor B handles Stage 5 (Proofread). This distributes the workload while ensuring fresh eyes at the final checkpoint.

Team Structure

The 27-person translation team is organized into specialized roles:

  • 18 Governors: Senior translators who review and approve translations at Stage 2 and Stage 5
  • 4 Terminology Specialists: Maintain the glossary and ensure term consistency
  • 3 Coordinators: Set up pipelines, assign models, and manage team allocation
  • 2 Final Reviewers: Provide the last quality gate before publication

Expected Outcome

With the structured pipeline and AI assistance, the team can run 4–6 parallel pipelines, increasing output from 1 book/year to an estimated 4–6 books/year.

System Design

  • FastAPI Backend: Handles all API routes, pipeline orchestration, and LLM calls
  • SQLite Database: Stores users, pipelines, glossary terms, and translation history
  • Multi-LLM Adapter: Unified interface to Claude, GPT-4, Gemini, and DeepSeek APIs (Claude active in public demo; others enabled when self-hosted)
  • Static Frontend: Vanilla HTML/JS served directly by FastAPI, no build step required
  • Docker Container: Single container deployment on Google Cloud Run. The public demo uses bring-your-own-key — visitors supply their own Anthropic key in the browser, so no keys are stored server-side

Setup Guide

Try the Live Demo

  • A free Anthropic API key — get one at console.anthropic.com (takes 2 minutes; Anthropic gives new accounts a small free credit)
  • Open the live app, paste your key into the API Key banner at the top, and log in as Zhang Wei (coordinator) to try the full pipeline

Self-Hosting (Optional)

Want to run it yourself, enable all four LLM providers, and disable the BYO-key prompt? Clone the repo and provide your own server-side keys via .env:

# Clone the repository git clone https://github.com/lyven81/bahai-translation-workbench.git cd bahai-translation-workbench # Install dependencies pip install -r requirements.txt # Set up environment variables cp .env.example .env # Add your API keys to .env (ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_API_KEY, DEEPSEEK_API_KEY) # Run the application python app.py
